## 2024-05-10 - Avoid useEffect for Derived State Filtering
**Learning:** Found a specific anti-pattern in the React code (`Explore.jsx` and likely others) where `useEffect` was used to map primary state (`models`) into derived state (`filteredModels`). This forces React to complete a render pass for the state change before firing the effect, which then sets the new state and causes a second render pass. This is particularly noticeable when users are typing fast into the search bar, causing jank. Additionally, child components inside these lists (`ModelCard.jsx`) were unmemoized, compounding the performance penalty of these double renders.
**Action:** Always compute derived state during the render cycle, using `useMemo` if the derivation (like list sorting/filtering) is expensive. Also, use `React.memo` on list item components (like `ModelCard`) that are pure functions of their props.
