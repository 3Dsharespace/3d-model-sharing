## 2024-05-24 - [Avoid useEffect for Derived State and Memoize Expensive Aggregations]
**Learning:** Using `useEffect` to compute derived state (like filtering or sorting lists) causes unnecessary double re-renders. A change in a dependency triggers the component to re-render, then the `useEffect` runs and updates state, which triggers a second re-render. Additionally, calculating aggregate stats using multiple chained array methods (`reduce`, `map`, `filter`) results in multiple O(N) passes through the array on every render.
**Action:** Use `useMemo` for derived state instead of `useEffect` + `useState`. This prevents the double re-render. Combine multiple O(N) array traversals for calculating statistics into a single O(N) `for...of` loop inside a `useMemo` hook to prevent redundant calculations on every render.
